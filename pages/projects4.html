Capstone Project 4

Predicting concrete strength

Finally at the halfway mark! We got to meet the great Python!(pun intended :)) Understanding the different libraries available was such a challenge. Troubleshooting proves to be a challenging feat as well! But with my strong mathematical background (I'm a Mechanical engineer yo!), understanding the topics for Machine Learning is much easier.

For this capstone, we had 9 days or so to complete the project. Imagine the pain and stress amounting to sleepless nights trying to get used to writing Python codes in order to churn out a proper report out.

Data Analytic is widely used in retail, finance, stocks industries and there are many use case used in ML. Since I was secretly enjoying the maths trivias/lessons in class, I decided to see if there were any science related data for ML use case (not for computer vision). It is just a matter of time before data analysis engulf each and every industry available on Earth :). Alas after a long search, I found a suitable dataset - Concrete strength!

Having had some work experience dealing with concrete/cement, I could tap into my experience for background knowledge. It is always better to have some domain knowledge when picking out the dataset. Cement, the main raw material, along with water, ash, sand, aggregates and other raw materials are mixed together to form concrete! In order to build bridges, buildings, shed etc, the strength required from the concrete differs. This cacn be done by mixing different quantities of the raw materials. As an engineer/operation team, having the correct mixing amount with the correct concrete strength would definitely reduce the time required and reduce the amount of possible wastage of the raw materials. In turn, increase profitability.

  • Business need: To predict the strength of concrete given the raw materials quantities
  • Tool used: Python (Pandas, Numpy, Matplotlib, Seaborn, Scipy), Jupyter Notebook, MS PowerPoint
  • Data file type: CSV file
  • Source: Concrete Strength Dataset (UCI Machine Learning Repository)
  • Problem model: Regression

Dataset was loaded into python where data cleaning, transformation and EDA was performed.Then, split the data into training and testing sets. ML models adopted were:

  1. Linear Regressor
  2. Ridge Regressor
  3. Lasso Regressor
  4. Random Forest Regressor
  5. SV Regressor
  6. KNN Regressor

Random Forest regressor came out tops performing extremely well with highest R2 score (0.873639) and lowest MSE (34.217022).
